# Widget Platform — MVP-прототип

Платформа встраиваемых виджетов с админкой и аналитикой (упрощённый аналог Elfsight).
Создаёшь виджет в админке → получаешь одну строку `<script>` → вставляешь на любой сайт.

## Что внутри
- **Loader `embed.js`** — ядро. Находит виджеты на чужом сайте, тянет конфиг, рендерит в **Shadow DOM** (чужой CSS не ломает виджет), шлёт аналитику.
- **5 типов виджетов:**
  - форма/подписка (поля, вебхук, сообщение об успехе);
  - поп-ап/баннер (триггеры: таймер / скролл / exit-intent, позиции, частота, картинка);
  - лента отзывов (сетка/список);
  - HTML/встраивание (любой HTML/CSS/JS в sandbox-iframe);
  - **плавающее видео** — кружок в углу экрана с автоплеем без звука; по клику разворачивается в модалку со звуком и контролами, крестик прячет виджет. Поддержка YouTube/Vimeo/MP4.
- **Админка** (`/admin/`) — список виджетов, конструктор с **живым превью**, загрузка картинок/видео, выбор **размещения** (в потоке страницы или закреплённым в любом углу — для формы, ленты и HTML), код вставки, пауза/дублирование, аналитика с фильтром по периоду, заявки с экспортом в CSV.
- **Бэкенд** (Node + Express) — публичные эндпоинты (конфиг, события, отправка формы, загрузка файла) и админские (CRUD, дубль, аналитика, CSV).
- **Демо-сайт** (`/demo-site.html`) — «чужой» сайт со всеми виджетами, встроенными одной строкой.
- **Хранилище** — JSON-файл (`data/db.json`), загруженные файлы — `public/uploads/`. В бою меняется на PostgreSQL/Supabase + объектное хранилище; интерфейс в `db.js` остаётся тем же.

## Запуск (нужен только Node.js 18+)

```bash
npm install        # установит express (единственная зависимость)
npm run seed       # создаст 3 демо-виджета и тестовую аналитику
npm start          # запустит сервер на http://localhost:3000
```

Затем открой:
- **Админка:**   http://localhost:3000/admin/
- **Демо-сайт:** http://localhost:3000/demo-site.html

### Что попробовать
1. На **демо-сайте** через пару секунд всплывёт поп-ап, видны форма и лента отзывов — всё подгружено одной строкой `<script src="/embed.js">`. Заметь: стили сайта намеренно ломающие (курсив, красные поля), но виджеты целы — спасибо Shadow DOM.
2. Отправь форму на демо-сайте → в **админке** на вкладке «Заявки» появится лид, в «Аналитике» вырастут показатели.
3. В админке измени настройки виджета и сохрани → обнови демо-сайт: виджет обновился **без правки кода на сайте** (магия серверного конфига).
4. Создай свой виджет кнопкой «+ Новый виджет», скопируй код вставки.

## Структура
```
server.js              Express: API + раздача статики
db.js                  Слой хранилища (JSON; легко заменить на Postgres)
seed.js                Демо-данные
public/
  embed.js             Loader + рантайм виджетов (Shadow DOM) — ЯДРО
  demo-site.html       Демонстрационный «чужой» сайт
  admin/               Админка (index.html, admin.css, admin.js)
data/db.json           Локальная база (создаётся при seed/работе)
```

## Важно про прототип (упрощения относительно боевой версии)
- **Авторизации нет** — админка открыта локально. В проде: вход + организации/роли (даёт Supabase Auth).
- **Хранилище — файл.** Для нагрузки → PostgreSQL (данные) + ClickHouse (события).
- **Рантайм на ванильном JS** в одном файле ради нулевой сборки. В проде → Preact + code-splitting по типам, минификация, раздача с CDN.
- **Соцсети** — лента работает на ручном/импортируемом контенте; подключение API Instagram/X — отдельный шаг (требует ключей и верификации).
- Защита домена и rate-limit заложены минимально — на проде усиливаются.
